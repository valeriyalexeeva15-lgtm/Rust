# Система визуального моделирования транспортных потоков

Консольное (TUI) приложение на Rust для моделирования и визуального анализа
транспортных потоков в дорожной сети. Логика моделирования отделена от интерфейса.

## Запуск

```
cargo run
```

Загрузка своей модели из файла:

```
cargo run -- scenarios/grid.json
```

## Управление

| Клавиша | Действие |
|---------|----------|
| Space | старт / пауза |
| S | стоп и формирование отчёта |
| R | сброс текущего сценария |
| 1 / 2 / 3 / 4 | сценарий: Базовый / Час пик / Перекрытие / Светофоры |
| + / − | скорость симуляции |
| , / . | уменьшить / увеличить интенсивность потока на ходу |
| Tab | выбрать сегмент (выделяется сиреневым) |
| C | перекрыть / открыть выбранный сегмент |
| L | сменить режим работы светофоров |
| стрелки | перемещение по карте |
| [ / ] | масштаб (zoom) |
| O | перезагрузить модель из файла |
| Q | выход |

## Цвета на карте

- зелёный — свободный участок;
- жёлтый — загруженный участок;
- красный — затор или перекрытый участок;
- белые точки — транспортные средства;
- E / X / + — въезд / выезд / перекрёсток.

## Структура

```
src/model.rs   модель сети, загрузка из JSON, валидация, ошибки
src/sim.rs     движок пошагового моделирования, сценарии, отчёт
src/main.rs    интерфейс (TUI), визуализация, управление
scenarios/     демонстрационные модели сети (JSON)
```

## Формат модели

Модель сети задаётся в JSON: узлы (`nodes`), сегменты (`segments`),
светофоры (`lights`), источники потока (`spawns`), типы ТС (`vehicle_kinds`).
При загрузке выполняется валидация структуры, ссылок, числовых параметров и
непротиворечивости. Ошибки возвращаются в структурированном виде и показываются
в интерфейсе, не приводя к аварийному завершению.
